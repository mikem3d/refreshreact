What is ReactJS

A JAVASCRIPT LIBRARY FOR BUILDING USER INTERFACES

Component based

By separating our UI into components, we can reduce code complexity, increase reusability, and separate concerns between components.

Virtual DOM

React abstracts away the DOM from you, giving a simpler programming model and better performance. React can also render on the server using Node, and it can power native apps using React Native.

Talk about how React creates an abstraction of the DOM, how costly it is to traverse the DOM, bind click events, etc. React does a "dirty" check of changes in the virtual DOM and only updates those elements which have changed.

Data Flow

React implements one-way reactive data flow which reduces boilerplate and is easier to reason about than traditional data binding.

There are two types of "model" data in React: props and state. It's important to understand the distinction between the two. Identify that state should drive the dynamic content in a component and has to live in the class

What about jQuery?

Most important take away:

In jQuery, you write event handlers which modify the DOM.
In React.js, you write event handlers which modify the state. And you write render() to reflect the current state.
